# оптимизация программной части SmartPC

---

## Хранение данных внутри приложения

На данный момент данные о сборках, комплектующих и услугах находятся непосредственно в коде:

```javascript
const builds = [
  {
    id: 1,
    name: "Smart Gamer RTX",
    price: 129990,
    cpu: "Ryzen 7 7800X3D",
    gpu: "RTX 4070 Super"
  }
];
```

сейчас для раннего сырого билда пойдет.

Однако при увеличении количества товаров поддержка станет сложнее. В дальнейшем данные рекомендуется хранить в базе данных и получать через API.

Преимущества:

* проще обновлять каталог;
* можно добавить панель администратора;
* не требуется изменять исходный код при добавлении новых товаров.

---

## Навигация внутри приложения

Сейчас переключение между страницами реализовано через состояние React:

```javascript
const [page, setPage] = useState("home");
```

Такое решение подходит для прототипа, но ограничивает развитие проекта.

Недостатки:

* отсутствуют отдельные URL страниц;
* невозможно открыть конкретный раздел по ссылке;
* возникают ограничения для SEO.

В будущих версиях проекта рекомендуется использовать React Router или Next.js Router.

## Конфигуратор ПК

На данный момент стоимость конфигурации рассчитывается достаточно просто:

```javascript
const total =
  cpu.price +
  gpu.price +
  ram.price +
  storage.price +
  cooling.price;
```

В рабочей версии конфигуратор можно расширить:

* проверкой совместимости сокетов;
* расчётом энергопотребления;
* подбором блока питания;
* предупреждением о несовместимых комплектующих.

Именно этот модуль может стать одним из ключевых преимуществ SmartPC.

## Фильтрация каталога

Сейчас фильтрация происходит на стороне клиента:

```javascript
const filtered = builds.filter(
  (b) =>
    budget === "all" ||
    (budget === "low" && b.price < 100000)
);
```

При небольшом количестве сборок это работает быстро.

Однако при росте каталога до сотен товаров нагрузка на браузер будет увеличиваться.

Для оптимизации можно перенести фильтрацию на серверную часть и загружать только необходимые данные.

## Работа с формами

После отправки формы пользователь получает сообщение:

```javascript
setSent(true);
```

На текущем этапе это демонстрация работы интерфейса.

В дальнейшем потребуется:

```javascript
await fetch("/api/order", {
  method: "POST",
  body: JSON.stringify(formData)
});
```

Это позволит сохранять заявки и передавать их менеджерам.

## Повторные рендеры компонентов

При каждом изменении состояния страницы React заново отрисовывает часть интерфейса.

Например:

```javascript
{filtered.map((b) => (
  <BuildCard
    key={b.id}
    build={b}
  />
))}
```

При большом каталоге это может привести к снижению производительности.

Для оптимизации можно использовать:

```javascript
export default React.memo(BuildCard);
```

или

```javascript
const filteredBuilds = useMemo(
  () => builds.filter(...),
  [budget]
);
```

## Масштабирование проекта

Сейчас большая часть логики находится в одном файле приложения.

По мере развития проекта рекомендуется разделить структуру на отдельные модули:

```text
src/
├── components/
├── pages/
├── services/
├── hooks/
├── api/
└── utils/
```

Такой подход значительно упростит поддержку и дальнейшее развитие SmartPC.
